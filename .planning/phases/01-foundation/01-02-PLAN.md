Create `backend/second_brain/ingestion/chunker.py`:

Define a dataclass `Chunk`:
```python
@dataclass
class Chunk:
    source_id: str    # UUID from sources table
    chunk_index: int
    content: str
    token_count: int
```

Define `chunk_text(text: str, source_id: str, target_tokens: int = 500, overlap_tokens: int = 50) -> list[Chunk]`:
- Use `tiktoken.get_encoding("cl100k_base")` to tokenize (this encoding works for all OpenAI-family models including OpenRouter).
- Split into sentences first using simple heuristic: split on `. `, `! `, `? `, `\n\n`. This avoids cutting mid-sentence.
- Accumulate sentences into chunks until token count reaches `target_tokens`. When a chunk is full, save it and start a new chunk that begins with the last `overlap_tokens` tokens of the previous chunk (for context continuity).
- Each `Chunk` gets a sequential `chunk_index` (0, 1, 2...).
- Minimum chunk size: skip chunks with fewer than 20 tokens.

Define `store_chunks_with_embeddings(chunks: list[Chunk], embed_provider: EmbeddingProvider, db: supabase.Client) -> int`:
- Returns count of chunks stored.
- Extract `[chunk.content for chunk in chunks]` into a list.
- Call `embed_provider.embed(chunk_texts)` in batches of 100 to get embeddings. The `embed` method returns `list[list[float]]`.
- For each chunk + embedding pair, insert into `chunks` table: `{source_id, chunk_index, content, token_count, embedding}`. The Supabase Python client accepts the embedding as a plain Python list of floats.
- Return total inserted count.

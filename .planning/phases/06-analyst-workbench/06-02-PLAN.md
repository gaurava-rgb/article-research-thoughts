# 06-02 Plan: Claims And Evidence

## Goal

Convert saved sources into analytical primitives that can be reused across timelines, suggestions, and concept maps.

## Scope

- add extraction flow for `entities`, `entity_aliases`, `source_entities`
- add extraction flow for `claims`, `claim_evidence`, and `claim_lenses`
- support claim-to-claim linking for at least:
  - `supports`
  - `contradicts`
  - `leads_to`
  - `amplifies`
  - `depends_on`
- record extraction provenance in `processing_runs`
- make extraction rerunnable and idempotent

## Out Of Scope

- full market sizing
- autonomous research loops
- final recommendation engine
- map rendering

## Backend Changes

- add an extraction module under `backend/second_brain/ingestion` or `backend/second_brain/analysis`
- define a stable extraction contract for one source
- persist evidence spans back to `chunks` or direct text offsets
- normalize entity names and basic alias handling
- add a small set of starter lenses, but keep the vocabulary extensible

## UI / API Changes

- add a source analysis payload or endpoint that returns:
  - extracted entities
  - extracted claims
  - evidence snippets
  - lenses
- show this on a source detail or debug inspection surface

## Testable Outcomes

- one source can be processed into entities, claims, and evidence rows
- rerunning extraction does not duplicate rows blindly
- evidence rows point to real source or chunk content
- contradictory or supporting claims can be linked
- extraction failures are visible in `processing_runs`

## Tangible User Value

- the user can open a source and see what the system thinks the article actually argued
- each claim can show "here is the text that justified this"
- this is the first moment where the app becomes inspectable instead of purely generative

## Exit Criteria

- extraction pipeline merged
- extraction tests added for at least one article-like source and one opinion/thesis source
- user can inspect a source's extracted claims with evidence

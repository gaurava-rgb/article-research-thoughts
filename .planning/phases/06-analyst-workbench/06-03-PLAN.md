# 06-03 Plan: Entity Dossier And Timeline

## Goal

Build the first analyst surface that answers:

- what is this company / product / market?
- what changed recently?
- why does the system believe this?

## Scope

- build an entity dossier view
- build a chronological claim timeline
- build a "what changed recently" summary
- build claim drilldown with evidence and source links
- support entity relationships such as ownership, competition, participation, dependency

## Out Of Scope

- generalized concept maps
- alerting
- auto-generated longform essays

## Backend Changes

- add query helpers over `entity_claim_timeline`
- add aggregation helpers for current thesis, recent changes, and linked entities
- support relation views like:
  - owns
  - competes_with
  - participates_in
  - depends_on

## UI / API Changes

- entity page or panel
- timeline view sorted by `event_at` or source publication time
- "why is this here?" drilldown for each claim
- relationship sidebar for products, markets, and competitors

## Testable Outcomes

- one entity page can be rendered from stored data only
- timeline ordering uses real dates and falls back predictably
- each displayed claim has traceable evidence
- contradictory claims are visibly distinguishable
- relationship queries return stable results for at least one company fixture

## Tangible User Value

- the user can inspect Google, YouTube, OpenAI, Microsoft, or another entity as a living dossier
- the user can stop asking the model to remember everything and instead browse the built timeline
- the system can answer "what changed?" with evidence instead of vibes

## Exit Criteria

- first non-chat analyst page shipped
- at least one real entity fixture shows timeline, claims, and relationships
- "why is this here?" is available for every visible claim

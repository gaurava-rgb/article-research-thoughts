# 06-01 Plan: Foundation Patch

## Goal

Patch the existing schema and Readwise ingest flow so the app can store structured analysis later without breaking the current product.

## Scope

- apply the additive schema changes from `docs/SPECS/post-v1-analyst-workbench.sql`
- enrich `sources` with `external_id`, `kind`, `tier`, `publisher`, `metadata`, `thread_key`, `parent_source_id`
- enrich `chunks` with structural fields such as `kind`, `section_label`, `speaker`, `page_number`, `start_char`, `end_char`
- add `processing_runs`
- keep Readwise as the only ingress
- classify each incoming source into at least a coarse `kind` and `tier`
- preserve backward compatibility for current search, chat, sync, and tests

## Out Of Scope

- entity extraction
- claim extraction
- timeline UI
- recommendation logic
- graph or map rendering

## Backend Changes

- patch `schema.sql`
- add migration notes or migration script for the new columns/tables
- patch `backend/second_brain/ingestion/readwise.py` to populate the new fields
- keep `readwise_id` working, but stop treating it as the long-term universal identity

## UI / API Changes

- minimal visible change only
- source/ingest surfaces should expose `kind`, `tier`, and `publisher` when available
- add a simple source detail payload if one does not already exist

## Testable Outcomes

- migration applies cleanly on an existing database with current data
- existing chat and retrieval tests still pass
- Readwise sync still inserts or updates sources successfully
- newly ingested sources have populated `kind`, `tier`, and `metadata`
- re-sync remains idempotent for unchanged sources

## Tangible User Value

- the reading inbox stops looking flat
- the user can tell whether an item is an article, thread, note, transcript, or similar
- the user can distinguish primary material from reporting, analysis, social, or personal notes

## Exit Criteria

- schema patch merged
- Readwise ingest patched and covered by tests
- one visible source detail improvement shipped
- no regression in the current chat path

# 06-04 Plan: Research Gaps And Follow-ups

## Goal

Suggest what the user should read next because it improves, validates, or challenges the current map.

## Scope

- reuse `insights` first for:
  - `coverage_gap`
  - `follow_up`
  - `counterpoint`
  - `watch`
- generate suggestions from entities, claims, claim links, and recent source activity
- explain each suggestion with a gap reason and triggering context
- keep the user in control of what gets read and saved into Readwise

## Out Of Scope

- ranking the entire internet
- auto-ingesting suggested articles
- consumer-style personalization feeds

## Backend Changes

- add suggestion generation job or on-demand endpoint
- detect missing primary sources, missing competitor context, stale timelines, or unsupported theses
- patch `insights` usage so suggestions link back to entities and claims

## UI / API Changes

- research queue or suggestions panel
- each suggestion shows:
  - why it exists
  - what entity or thesis triggered it
  - what kind of gap it fills
  - suggested query, source, or URL

## Testable Outcomes

- system can generate at least one suggestion type from stored claims
- each suggestion has explicit provenance
- dismissing or marking seen works without affecting raw source data
- suggestion logic does not require new connectors

## Tangible User Value

- the app starts helping the user deepen the map instead of only summarizing what is already saved
- the user gets prompts like "you have three analysis pieces but no primary source on this topic"
- this is the first step toward an analyst copilot instead of a passive archive

## Exit Criteria

- suggestion types implemented and inspectable
- at least one user-facing suggestion surface shipped
- every suggestion can answer "why are you showing me this?"

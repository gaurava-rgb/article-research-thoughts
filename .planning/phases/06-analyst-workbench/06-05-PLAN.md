# 06-05 Plan: Concept Map And Ripple View

## Goal

Render an annotated concept map and ripple view from stored analytical primitives.

## Scope

- render nodes from entities, claims, markets, products, and insights
- render edges from entity relationships and claim links
- allow grouping by entity, product, market, thesis, lens, leading indicator, and lagging outcome
- allow drilldown on every node and edge
- add a ripple-oriented lens for "if this is true, what else moves?"

## Out Of Scope

- free-form LLM graph generation with no backing data
- exhaustive ecosystem mapping for every source on day one

## Backend Changes

- add graph/query assembly from stored primitives
- define node/edge confidence and support counts
- support subgraph retrieval for one entity or one thesis

## UI / API Changes

- concept map view
- ripple view
- node detail drawer
- edge detail drawer

## Testable Outcomes

- every rendered node maps to a stored record
- every rendered edge maps to a stored relation or claim link
- clicking a node shows supporting and contradictory evidence
- clicking an edge shows why the connection exists
- one entity-centered subgraph can render without model calls

## Tangible User Value

- the user gets the "annotated mind map" version of the product
- the map is not magical but inspectable
- the system can surface intermediate causes between source facts and high-level conclusions

## Exit Criteria

- concept map ships with evidence drilldown
- ripple exploration works for at least one entity or thesis fixture
- the map can answer "why is this bubble here?" and "why are these two things connected?"

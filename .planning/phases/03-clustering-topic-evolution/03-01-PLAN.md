```sql
-- Phase 3 schema additions
-- topics.centroid_embedding: running centroid of all member source embeddings
ALTER TABLE topics ADD COLUMN IF NOT EXISTS centroid_embedding vector(1536);

-- match_topic: SQL function to find best-matching topic for a source embedding
-- Returns (topic_id, topic_name, similarity) sorted by cosine similarity DESC
-- 1 - (centroid_embedding <=> query_embedding) gives similarity (not distance)
CREATE OR REPLACE FUNCTION match_topic(
  query_embedding vector(1536),
  match_threshold float DEFAULT 0.65,
  match_count int DEFAULT 1
)
RETURNS TABLE (topic_id uuid, topic_name text, similarity float)
LANGUAGE sql STABLE AS $$
  SELECT
    id AS topic_id,
    name AS topic_name,
    1 - (centroid_embedding <=> query_embedding) AS similarity
  FROM topics
  WHERE centroid_embedding IS NOT NULL
    AND 1 - (centroid_embedding <=> query_embedding) >= match_threshold
  ORDER BY centroid_embedding <=> query_embedding
  LIMIT match_count;
$$;
```

"""
ingestion/clustering.py — Topic assignment for the Second Brain

This module handles automatic article-to-topic assignment on ingestion (STR-01),
topic summary regeneration when new articles join (STR-02), and date-filtered
topic source retrieval for temporal queries (STR-03).

Algorithm overview:
  1. Load the stored whole-source embedding from `sources.source_embedding`.
     This vector is already written during ingestion and is the source-level
     vector of record for Phase 3 topic work.
  2. Compare against all existing topic centroids using the match_topic SQL
     function (cosine similarity >= SIMILARITY_THRESHOLD = 0.65).
  3. If a match: assign source to that topic.
     If no match: ask LLM to name a new topic, create it, assign source to it.
  4. After any assignment: update the topic centroid (mean of all member sources).
  5. After all assignments in a sync batch: regenerate summaries for changed topics
     (called once per changed topic, NOT once per article — avoids O(N) LLM calls).

Usage (called from cli.py after chunking):
    from second_brain.ingestion.clustering import assign_topics_to_source

    topic_id = assign_topics_to_source(source_id, db, embed_provider, llm_provider)

Why LLM-guided centroid matching instead of HDBSCAN?
  HDBSCAN cannot incrementally add a single document to an existing cluster
  without a full re-cluster (confirmed: BERTopic docs + arxiv 2601.20680).
  Full re-cluster discards source_topics assignments and produces unstable
  topic IDs. Centroid threshold approach is O(N_topics) per article and
  preserves all prior assignments.
"""

<decisions_made>

- **Clustering approach:** LLM-guided centroid matching (not HDBSCAN/BERTopic) — HDBSCAN cannot incrementally add a single document without full re-cluster, which breaks stable topic IDs
- **Similarity threshold:** Start at 0.65 cosine similarity; tune after first real sync by logging similarity scores
- **Schema additions needed:** `sources.embedding vector(1536)` (source-level avg of chunk embeddings) + `topics.centroid_embedding vector(1536)` + optional `match_topic` SQL function
- **Batched summary regen:** Changed topic IDs are collected in a set during sync; LLM is called once per changed topic at the end (not once per article)
- **Temporal fallback:** `COALESCE(published_at, ingested_at)` for articles without publication dates
- **Topics API in Phase 3:** `GET /api/topics` added now to avoid Phase 4 plan backtracking
- **TDD approach:** Plan 03-01 writes 7 failing tests first; Plan 03-02 drives them to GREEN
</decisions_made>

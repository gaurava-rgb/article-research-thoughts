<decisions_made>

- **Clustering approach:** LLM-guided centroid matching (not HDBSCAN/BERTopic) — HDBSCAN cannot incrementally add a single document without full re-cluster, which breaks stable topic IDs
- **Similarity threshold:** Start at 0.65 cosine similarity; tune after first real sync
- **Existing prerequisite already shipped:** `sources.source_embedding vector(1536)` is already in schema and is written during ingestion
- **Remaining schema additions needed:** `topics.centroid_embedding vector(1536)` + `match_topic` SQL function
- **Batched summary regen:** Changed topic IDs collected in a set during sync; LLM called once per changed topic at the end
- **Temporal fallback:** `COALESCE(published_at, ingested_at)` for articles without publication dates
- **Topics API in Phase 3:** `GET /api/topics` added now to avoid Phase 4 backtracking
- **TDD approach:** Plan 03-01 writes 7 failing tests first; Plan 03-02 drives them to GREEN
